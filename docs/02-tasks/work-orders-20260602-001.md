### WO-01: 매물 선택 시 지도 전체 재초기화 버그

**파일**: `widgets/map-view/ui/kakao-map.tsx`  
**담당**: frontend-dev

#### 문제

```typescript
// kakao-map.tsx:43-111
useEffect(() => {
  // ... 지도 초기화, 마커 생성 전부 여기에 있음
}, [isMapLoaded, properties, selectedProperty, setSelectedProperty]);
//                           ^^^^^^^^^^^^^^^^
// selectedProperty가 바뀔 때마다 지도 전체가 재초기화됨
// → 마커 전부 지우고 다시 그림 (깜빡임 + 성능 저하)
```

#### 해결 방법

`useEffect`를 두 개로 분리:

```typescript
// Effect 1: 지도 + 마커 초기화 (최초 1회)
useEffect(() => {
  if (!isMapLoaded || !mapRef.current) return;

  const map = new window.kakao.maps.Map(mapRef.current, mapOption);
  mapInstanceRef.current = map;  // ref로 map 인스턴스 저장

  const markers = properties.map((property) => {
    // 마커 생성 로직...
    return marker;
  });
  markersRef.current = markers;

  return () => {
    markers.forEach(m => m.setMap(null));
  };
}, [isMapLoaded, properties]);  // selectedProperty 제거

// Effect 2: 선택된 매물로 카메라 이동만
useEffect(() => {
  if (!mapInstanceRef.current || !selectedProperty) return;
  const position = new window.kakao.maps.LatLng(selectedProperty.lat, selectedProperty.lng);
  mapInstanceRef.current.setCenter(position);
  mapInstanceRef.current.setLevel(4);
}, [selectedProperty]);  // 카메라 이동만
```

**추가**: `mapInstanceRef`, `markersRef`를 `useRef`로 선언

### WO-04: 루트 페이지 'use client' 제거

**파일**: `app/page.tsx`  
**담당**: frontend-dev

#### 문제

```typescript
// app/page.tsx:1
'use client';  // ← App Router에서 page.tsx는 서버 컴포넌트여야 함
```

Kakao 스크립트 로딩과 뷰 전환 상태 구독을 위해 'use client'가 붙어 있으나,
이를 분리된 클라이언트 컴포넌트로 빼면 해결 가능.

#### 해결 방법

```typescript
// app/page.tsx (서버 컴포넌트로 유지)
import { ClientLayout } from '@/widgets/client-layout/ui/client-layout';
export default function HomePage() {
  return <ClientLayout />;
}

// widgets/client-layout/ui/client-layout.tsx (클라이언트 컴포넌트)
'use client';
export function ClientLayout() {
  const { currentView } = useAppStore();
  useEffect(() => { loadKakaoMapScript(...); }, []);
  return (
    <div className="h-screen flex flex-col overflow-hidden">
      <Header />
      {currentView === 'map' ? <MapView /> : <div className="flex-1 overflow-auto"><LoanCalculator /></div>}
    </div>
  );
}
```

### WO-05: Primary 색상 통일

**파일**: `app/globals.css`, `widgets/calculator-view/ui/loan-calculator.tsx`  
**담당**: frontend-dev

#### 문제

`globals.css`의 `--primary`는 near-black이지만 `loan-calculator.tsx`에 하드코딩된 파란색 사용:
- `text-blue-600`, `bg-blue-50`, `border-blue-200`
- `from-blue-500 to-indigo-600` (그라디언트 카드)
- `text-blue-100`, `text-blue-600` (금리 표시)

#### 해결 방법 (A안 권장): primary를 파란색으로 변경

```css
/* globals.css — primary를 blue로 통일 */
:root {
  --primary: oklch(0.546 0.196 258.316);          /* blue-600 */
  --primary-foreground: oklch(0.985 0 0);          /* white */
}
.dark {
  --primary: oklch(0.707 0.165 258.316);          /* blue-400 */
  --primary-foreground: oklch(0.145 0 0);
}
```

그 후 `loan-calculator.tsx`의 `text-blue-600` → `text-primary` 등으로 치환.

### WO-07: Kakao SDK 로딩 방식 개선

**파일**: `shared/lib/kakao-map.ts`, `app/page.tsx` (또는 분리된 ClientLayout)  
**담당**: frontend-dev

#### 문제

현재 `setInterval` 폴링으로 SDK 로드 감지:

```typescript
// kakao-map.tsx:30-38
const checkKakao = setInterval(() => {
  if (window.kakao && window.kakao.maps) {
    clearInterval(checkKakao);
    window.kakao.maps.load(() => setIsMapLoaded(true));
  }
}, 100);
```

#### 해결 방법

`app/layout.tsx`에서 Next.js `<Script>` 컴포넌트 사용:

```typescript
// app/layout.tsx
import Script from 'next/script';

export default function RootLayout({ children }) {
  return (
    <html lang="ko">
      <body>
        <Script
          src={`//dapi.kakao.com/maps/sdk.js?appkey=${process.env.NEXT_PUBLIC_KAKAO_MAP_API_KEY}&autoload=false`}
          strategy="afterInteractive"
          onLoad={() => window.kakao.maps.load(() => {
            window.dispatchEvent(new Event('kakao-map-ready'));
          })}
        />
        {children}
      </body>
    </html>
  );
}

// kakao-map.tsx — useEffect에서 이벤트 리스닝
useEffect(() => {
  const onReady = () => setIsMapLoaded(true);
  if (window.kakao?.maps) { onReady(); }
  else { window.addEventListener('kakao-map-ready', onReady); }
  return () => window.removeEventListener('kakao-map-ready', onReady);
}, []);
```

### WO-10: 대출금액 입력 최솟값 검증

**파일**: `widgets/calculator-view/ui/loan-calculator.tsx:192`  
**담당**: frontend-dev

```typescript
// 현재
<Input type="number" value={loanAmount} onChange={(e) => setLoanAmount(Number(e.target.value))} />

// 수정: min=0, 음수 방어
<Input
  type="number"
  value={loanAmount}
  min={0}
  max={selectedProduct.maxAmount}
  onChange={(e) => {
    const val = Math.max(0, Math.min(Number(e.target.value), selectedProduct.maxAmount));
    setLoanAmount(val);
  }}
/>
```
